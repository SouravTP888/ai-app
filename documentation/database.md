# Database Schema - AI LMS Automation Engine

This document details the data structures and schemas used in the system, covering both MongoDB Mongoose schemas and the local JSON file database fallback.

---

## Storage Modes

The system operates in a hybrid storage mode controlled by the availability of MongoDB:
1. **MongoDB Mode**: If a connection to `mongodb://127.0.0.1:27017/ai-lms` (or `MONGO_URI` env) is successfully established, the system uses Mongoose schemas.
2. **Local JSON Database Fallback Mode**: If MongoDB is offline, the backend utilizes `backend/src/utils/dbService.js` to read and write records into simple JSON files located under `backend/data/` (e.g. `users.json`, `courses.json`, `progress.json`, `learningpaths.json`). This ensures the app is immediately runnable without database administration.

---

## Data Models

### 1. User
Represents registered students and administrators.
- **Attributes**:
  - `name`: String (required)
  - `email`: String (required, unique)
  - `password`: String (hashed)
  - `role`: String (`student` or `admin`)
  - `selectedTrack`: String (`AI Engineer`, `Full Stack Developer`, `Data Scientist`, `Cyber Security Specialist`)
  - `skillLevel`: String (`Beginner`, `Intermediate`, `Advanced`)
  - `interests`: Array of Strings
  - `createdAt`: Date

### 2. Course
Catalog of modules available for study.
- **Attributes**:
  - `title`: String (required, unique)
  - `description`: String (required)
  - `category`: String (matches student tracks, e.g. `AI Engineer`)
  - `difficulty`: String (`Beginner`, `Intermediate`, `Advanced`)
  - `duration`: Number (in hours)
  - `modules`: Array of objects:
    - `_id`: Auto-generated unique identifier
    - `title`: String
    - `description`: String
    - `duration`: Number (in minutes)
  - `createdAt`: Date

### 3. Learning Path (Roadmap)
A student's personalized journey map generated by the AI Engine.
- **Attributes**:
  - `userId`: Schema.Types.ObjectId (references User)
  - `recommendedCourses`: Array of ObjectIds (references Course)
  - `roadmapStages`: Array of stage objects:
    - `phase`: String (e.g. "Phase 1: Foundations")
    - `title`: String (e.g. "Python Fundamentals")
    - `description`: String
    - `topics`: Array of Strings
    - `courses`: Array of ObjectIds (references Course)
  - `createdAt`: Date

### 4. Progress
Tracks lesson metrics and completion values per course.
- **Attributes**:
  - `userId`: Schema.Types.ObjectId (references User)
  - `courseId`: Schema.Types.ObjectId (references Course)
  - `completedModules`: Array of Module IDs (representing completed lessons)
  - `completionPercentage`: Number (0 to 100, dynamically calculated based on completedModules / course.modules.length)
  - `status`: String (`Not Started`, `In Progress`, `Completed`)
  - `lastAccessed`: Date

## Pre-seeded Data

When the database (either MongoDB or the JSON file fallback) is empty, the server automatically seeds the following items:
- **Default Users**:
  - **Admin**: `admin@eduflick.ai` / `adminpassword`
  - **Student**: `john@student.com` / `studentpassword`
- **Default Catalog**:
  - 12 core courses spanning the career tracks (e.g. "Python Fundamentals for AI", "Machine Learning Essentials", "Deep Learning & Neural Networks", "HTML, CSS & JavaScript Fundamentals", etc.), fully complete with modules and lessons.
